DRAFT :: Docker Swarm试用

18 Feb, 2017 · Read in about 1 min · (159 Words)
dev work

关于Docker Swarm

Docker Swarm(swarm mode)是 Docker .inc 于docker 1.12版本中集成的集群管理解决方案。

配置

机器配置

主机名 IP地址 主从类型
master2 192.168.8.2 Master
master3 192.168.8.3 Master
master4 192.168.8.4 Master
worker5 192.168.8.5 Worker
worker6 192.168.8.6 Worker
worker7 192.168.8.7 Worker

版本：

OS:     Ubuntu 16.04 amd64
Docker: docker v1.13.1

配置Swarm Cluster

在master2主机上，初始化一个swarm集群

docker swarm init

基本使用

  1. 网络创建

    # 创建一个可挂载的overlay类型网络inetw
docker network create -d overlay --attachable inetw

  2. 新建服务

    # 创建一个拥有三个实例的nginx服务，将其绑定在网络inetw下面，将其端口号80映射到主机80端口上
docker service create --name tt --replicas 3 --network inetw -p 80:80 nginx

  3. 更新服务

    # 更新名为tt的nginx服务，强制让其进行更新（即使容器的镜像未发生变化）
docker service update --with-registry-auth --force --image nginx tt
# --with-registry-auth 是因为docker 1.13的bug，参见：https://github.com/docker/docker/issues/30703

配合Compose使用

Docker Swarm mode并不能直接使用Docker Compose进行管理使用。 但是，Docker在1.13版本中提出了一个stack的概念，提供类似Compose实现的功能。

Docker stack配置文件示例：

version: '3'

services:
  api:
    image: hub.private.com/dev/api
    environment:
      - DEBUG=true
    ports:
      - 8060:8090
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure
  sso:
    image: hub.private.com/dev/sso
    ports:
      - 8068:8080
    depends_on:
      - api
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure
  console:
    image: hub.private.com/dev/console
    ports:
      - 8061:3000
    depends_on:
      - api
    deploy:
      replicas: 2
      restart_policy:
        condition: on-failure
  ldap:
    image: hub.private.com/dev/ldap
    ports:
      - 8063:389
    deploy:
      replicas: 2
      restart_policy:
        condition: on-failure

其它
